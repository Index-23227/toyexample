# Step 0: DB가 뭔가요? (20분)

> 엑셀로 잘 하고 있는데, 왜 DB라는 걸 쓰는 걸까요?
> 이 단계에서는 **직접 코딩하지 않습니다**. 개념만 잡으면 됩니다.

---

## 엑셀과 DB, 뭐가 다른가요?

여러분은 이미 week04에서 엑셀 자동화를 해봤습니다.
법인 파일 5개를 합치고, 경비를 정리했죠.

그런데 현실에서는 이런 상황이 생깁니다:

> "법인이 20개로 늘었어요. 엑셀 파일이 60개인데 합칠 수 있나요?"
> "팀장님이 법인별 매출을 **웹에서 바로 보고 싶대요.**"
> "다른 팀원이 같은 파일을 동시에 열었는데 충돌이 나요."

이때 DB가 필요합니다.

---

## 한 장 비교

| | 엑셀 | DB (데이터베이스) |
|--|------|------------------|
| 파일 | .xlsx 파일 하나하나 관리 | 하나의 DB 안에 테이블 여러 개 |
| 찾기 | Ctrl+F, 필터 | SQL 한 줄로 정확히 검색 |
| 합치기 | 복사-붙여넣기 or VBA | JOIN 한 줄 |
| 동시 작업 | 한 명만 편집 가능 | 여러 명 동시 가능 |
| 데이터 양 | 100만 행 한계 | 수억 행 OK |
| 웹 연결 | 안 됨 | Flask 등으로 바로 연결 |

---

## SQL이 뭔가요?

SQL = **DB에게 시키는 명령어**. 엑셀의 "필터"와 비슷한 역할입니다.

```
엑셀: "미국" 필터 → 미국 데이터만 보기
SQL:  SELECT * FROM monthly_sales WHERE corp_code = 'US01';
```

```
엑셀: 피벗 테이블 → 법인별 합계
SQL:  SELECT corp_code, SUM(amount) FROM monthly_sales GROUP BY corp_code;
```

**하지만 여러분이 SQL을 쓸 필요는 없습니다.**
CLAUDE.md에 DB 구조를 적어두면, AI가 SQL을 대신 써줍니다.

---

## 오늘 쓸 DB 구조

week04에서 다룬 법인 매출 데이터가 이미 DB에 들어있습니다:

```
sales.db (SQLite 데이터베이스)
├── corporations    → 법인 정보 (미국법인, 일본법인, ...)
├── monthly_sales   → 월별 매출 (법인별 1~3월 매출액)
└── exchange_rates  → 환율 (USD 1,350원, JPY 9.2원, ...)
```

> **SQLite란?** 파일 하나짜리 가벼운 DB입니다.
> MySQL 같은 서버 설치 없이, Python에 기본 내장되어 있어 바로 쓸 수 있습니다.
