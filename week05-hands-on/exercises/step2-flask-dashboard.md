# Step 2: Flask 대시보드 만들기 (25분)

> 터미널에서 보던 결과를 **웹 브라우저**에서 보이게 만듭니다.
> Flask = Python으로 웹 페이지를 만드는 도구. 여러분은 만들지 않습니다 — AI가 만듭니다.

---

## Flask가 뭔가요?

week04 Step 3에서 HTML 보고서를 만들어본 적 있죠?
그때는 **정적 HTML** — 파일을 더블클릭해서 열었습니다.

Flask는 **동적 웹 페이지** — 서버가 DB에서 데이터를 꺼내서 실시간으로 보여줍니다.

```
정적 HTML:  파일 만들기 → 더블클릭 → 항상 같은 내용
Flask:      서버 실행 → URL 접속 → DB에서 최신 데이터를 가져와서 보여줌
```

> DB에 새 데이터가 추가되면? 새로고침만 하면 바로 반영!

---

## 작게 시작하기 — 메인 페이지

### AI에게 시켜보기

```
data/sales.db에 연결해서 법인별 매출 현황을 보여주는
Flask 웹 앱을 만들어줘.

요구사항:
1. 메인 페이지(/)에 법인별 매출 합계 표를 보여줘
2. corporations와 monthly_sales를 JOIN해서 법인명으로 표시
3. exchange_rates 환율로 원화 환산 금액도 같이 보여줘
4. 법인명 가나다순 정렬
5. 금액은 천 단위 콤마 표시
6. HTML 테이블에 간단한 CSS 스타일 적용

파일 구조:
- app.py (Flask 앱)
- templates/index.html (메인 페이지)
```

### 실행하기

AI가 파일을 만들어주면, 터미널에서:

```
python app.py
```

이런 메시지가 나옵니다:
```
 * Running on http://127.0.0.1:5000
```

**브라우저에서 http://127.0.0.1:5000 접속!**

## 만약 서버가 안 뜨면?

AI에게 시키기 전에, 일단 Flask가 설치되어있는지 확인:

```
python -m pip install flask
```

### 자주 나는 에러

| 에러 | 원인 | 해결 |
|------|------|------|
| `ModuleNotFoundError: No module named 'flask'` | Flask 미설치 | `python -m pip install flask` |
| `Address already in use` | 이미 서버가 켜져있음 | 터미널에서 `Ctrl+C` 후 다시 실행 |
| `TemplateNotFound` | HTML 파일 위치가 다름 | `templates/` 폴더에 파일이 있는지 확인 |
| `OperationalError: no such table` | DB 미생성 | `python data/create_db.py` 실행 |
| 페이지가 안 열림 | 브라우저 주소 | `http://127.0.0.1:5000` 정확히 입력 |

**에러가 나면 당황하지 말고 에러 메시지를 AI에게 그대로 보여주세요.**

```
python app.py를 실행했더니 이런 에러가 나:
[에러 메시지 붙여넣기]
고쳐줘.
```
