## 공통 규칙

### 파일명 규칙
| 용도 | 패턴 | 실제 예시 |
|------|------|-----------|
| 경비 정리 결과 | `[재경]경비정리_YYYYMM_v{N}.xlsx` | `[재경]경비정리_202603_v1.xlsx` |
| 법인 원본 | `법인_{법인코드}_{국가명}.xlsx` | `법인_US01_미국.xlsx` |
| 법인 통합 결과 | `법인매출통합.xlsx` | |
| 외화 채권 원장 | `외화채권잔액.xlsx` | |

### 데이터 형식
- **날짜**: `YYYY-MM-DD` (일 단위) 또는 `YYYY-MM` (월 단위)
- **금액**: 정수, 소수점 없음, 천 단위 구분 없이 숫자 저장
- **통화코드**: ISO 4217 — `USD`, `JPY`, `CNY`, `EUR`, `VND`
- **법인코드**: `{국가2자리}{숫자2자리}` — `US01`, `JP01`, `CN01`, `DE01`, `VN01`

### 정렬 원칙
- 부서: **영업1팀 → 영업2팀 → 관리팀** 순서 (가나다순 아님, 사내 고정 순서)
- 법인: **가나다순** (독일 → 미국 → 베트남 → 일본 → 중국)
- 날짜: 오름차순
- **합계 행은 항상 마지막**

### 부서 목록 (고정)
`영업1팀`, `영업2팀`, `관리팀`

### 경비 항목 목록 (현재 기준)
`교통비`, `식대`, `사무용품`, `접대비`

### 라이브러리
- Excel 처리: `openpyxl`
- DB: `sqlite3` (Python 내장)
- 웹: `flask` (Part C에서 사용)

**파일**: `data/경비내역.xlsx` (시트명: `Sheet`)

| 컬럼 | 타입 | 예시 |

| 결제방법 | TEXT | `법인카드`, `개인카드` |

실제 데이터 (15건):
```

2026-03-07  관리팀   사무용품  45000   법인카드
2026-03-09  영업2팀  식대      18000   법인카드
2026-03-11  영업1팀  접대비   120000   법인카드
2026-03-13  관리팀   교통비    12000   개인카드
2026-03-15  영업2팀  교통비    25000   법인카드

```

### 비즈니스 규칙

#### 경비 구분
| 구분 | 해당 항목 |
|------|-----------|
| **일반경비** | `교통비`, `식대`, `사무용품` (+ 새 항목은 기본 일반경비) |
| **특별경비** | `접대비` (항목명으로 판단, 금액 무관) |

#### 비고 자동 생성 (우선순위 주의)
| 조건 | 비고 값 |
|------|---------|
| `접대비` + `개인카드` | `※ 팀장 사전승인` ← **이것만 기재, 정산대상 문구 중복 X** |
| `접대비` + `법인카드` | `※ 증빙 첨부 필수` |
| `개인카드` (접대비 외) | `개인카드 - 정산대상` |
| 그 외 | 비워둠 (`None`) |

> 핵심: 접대비+개인카드 → "※ 팀장 사전승인" **단독 기재**. "개인카드 - 정산대상"과 중복 X.

#### 개인카드 정산
- 결제방법이 `개인카드`인 건은 모두 별도 시트에 추출
- 정산상태 초기값: `미정산`

### 출력

**파일명**: `data/[재경]경비정리_202603_v1.xlsx`

**시트 3개, 순서 고정: 경비총괄표 → 전체내역 → 개인카드_정산**

#### 시트 1: `경비총괄표`
부서별 일반/특별 경비 요약 피벗. 부서 순서는 **영업1팀 → 영업2팀 → 관리팀 → 합계**.

| 구분 | 건수 | 일반경비 | 특별경비 | 합계 |
|------|------|----------|----------|------|
| 영업1팀 | 6 | 92000 | 270000 | 362000 |
| 영업2팀 | 5 | 106000 | 95000 | 201000 |
| 관리팀 | 4 | 146000 | 0 | 146000 |
| **합계** | **15** | **344000** | **365000** | **709000** |

#### 시트 2: `전체내역`
원본에 `경비구분`, `비고` 컬럼 추가. 일자 오름차순 정렬.

| 컬럼 | 원본/추가 |
|------|-----------|
| 일자 | 원본 |
| 부서 | 원본 |
| 항목 | 원본 |
| 금액 | 원본 |
| 결제방법 | 원본 |
| 경비구분 | **추가** — `일반경비` 또는 `특별경비` |
| 비고 | **추가** — 위 규칙 적용 |

기대 결과 예시:
```
2026-03-05  영업1팀  교통비   28000  개인카드  일반경비  개인카드 - 정산대상
2026-03-11  영업1팀  접대비  120000  법인카드  특별경비  ※ 증빙 첨부 필수
2026-03-21  영업2팀  접대비   95000  개인카드  특별경비  ※ 팀장 사전승인
```

#### 시트 3: `개인카드_정산`
개인카드 결제건만 추출. 일자 오름차순.

| 일자 | 부서 | 항목 | 금액 | 정산상태 |
|------|------|------|------|----------|
| 2026-03-05 | 영업1팀 | 교통비 | 28000 | 미정산 |
| 2026-03-13 | 관리팀 | 교통비 | 12000 | 미정산 |
| 2026-03-21 | 영업2팀 | 접대비 | 95000 | 미정산 |
| 2026-03-27 | 영업2팀 | 교통비 | 31000 | 미정산 |

### 입력

## DB 스키마 (어드밴스)

**SQLite**: `data/sales.db`

### 테이블: `corporations`
| 컬럼 | 타입 | 키 | 예시 |
|------|------|-----|------|
| corp_code | TEXT | PK | `US01` |
| corp_name | TEXT | | `미국법인` |
| country | TEXT | | `미국` |
| currency | TEXT | | `USD` |

### 테이블: `monthly_sales`
| 컬럼 | 타입 | 키 | 예시 |
|------|------|-----|------|
| id | INTEGER | PK AUTO | |
| corp_code | TEXT | FK → corporations | `US01` |
| month | TEXT | | `2026-01` |
| amount | REAL | | `125000.0` |
| note | TEXT | | `신규 거래처` |

### 테이블: `exchange_rates`
| 컬럼 | 타입 | 키 | 예시 |
|------|------|-----|------|
| currency | TEXT | PK(복합) | `USD` |
| rate_date | TEXT | PK(복합) | `2026-03-31` |
| rate | REAL | | `1350.0` |

### 주요 쿼리 패턴

```sql
-- 법인별 매출 합계 (현지통화)
SELECT c.corp_name, c.currency, SUM(s.amount) AS total
FROM monthly_sales s
JOIN corporations c ON s.corp_code = c.corp_code
GROUP BY c.corp_name;

-- 원화 환산 매출
SELECT c.corp_name, s.month, s.amount,
       ROUND(s.amount * e.rate) AS krw_amount
FROM monthly_sales s
JOIN corporations c ON s.corp_code = c.corp_code
JOIN exchange_rates e ON c.currency = e.currency
WHERE e.rate_date = '2026-03-31';
```

## 작업 시 반드시 지킬 것

1. **접대비 = 항상 특별경비**. 금액 무관, 항목명으로 판단.
2. **접대비 + 개인카드 → 비고는 `※ 팀장 사전승인` 단독 기재**. `개인카드 - 정산대상` 중복 X.
3. **법인 파일 합칠 때 법인코드/법인명은 파일명에서 파싱**. 파일 내부에는 없음.
4. **법인별합계는 현지 통화 기준**. 원화 환산 아님.
5. **외화 환산 금액은 반올림 정수**.
6. **시트 순서 고정**:
   - 경비: `경비총괄표` → `전체내역` → `개인카드_정산`
   - 법인: `통합데이터` → `법인별합계` → `월별피벗`
7. **합계 행의 구분 컬럼에는 반드시 `합계`로 표기**.
8. **부서 정렬**: 영업1팀 → 영업2팀 → 관리팀 (사내 고정 순서).
9. **법인 정렬**: 가나다순 (독일 → 미국 → 베트남 → 일본 → 중국).
10. **결과 파일은 모두 `data/` 폴더에 저장**.
