# Week 07 — "이런 것도 됩니다" 바이브코딩 쇼케이스

> 강사가 Claude Code에 **자연어로 시키는 것만으로** 결과를 만들어냅니다.
> 학생이 "나도 저렇게 하면 되는구나"를 체감하는 게 목적입니다.
> 스크립트는 직접 실행하지 않습니다. **Claude에게 말하면 Claude가 만들고 실행합니다.**

---

## 시연 방법

1. Claude Code를 열고
2. 아래 프롬프트를 **그대로 복붙**하고
3. Claude가 코드를 만들고 실행하는 걸 학생에게 보여주세요

> `demos/` 폴더의 스크립트는 **참고 정답**입니다. Claude가 비슷한 결과를 만들지 못할 때 강사용으로 사용합니다.

---

## 데모 1: 두 엑셀 비교 (5분)

> **학생이 느낄 것**: "매달 눈으로 비교하던 걸 한마디로 끝내네"

### Claude에게 이렇게 말하세요

```
week07-hands-on/demos/output/ 폴더에
3월_매출.xlsx와 4월_매출.xlsx 샘플 데이터를 만들어줘.
법인 8개의 매출 금액이 들어있고, 4월에는 몇 개 법인의 금액이 바뀌어있어.

그다음 두 파일을 비교해서:
- 금액이 바뀐 법인만 추출
- 이전 금액, 이후 금액, 차이, 변동률을 계산
- 결과를 "비교결과.xlsx"로 저장 (증가는 빨간색, 감소는 초록색)
```

### 기대 결과
- 비교 결과 엑셀이 생성됨
- 변경된 법인만 추출, 증감률 자동 계산
- 색칠까지 완료

---

## 데모 2: PDF → 엑셀 추출 (5분)

> **학생이 느낄 것**: "세금계산서 10장 손으로 옮기던 거, 이걸로 되는구나"

### Claude에게 이렇게 말하세요

```
세금계산서 3장의 텍스트 내용을 시뮬레이션해서
공급자, 사업자번호, 작성일자, 품목, 공급가액, 세액, 합계를 추출한 뒤
엑셀로 정리해줘. 합계 행도 넣고, 숫자는 천단위 콤마 서식.
week07-hands-on/demos/output/ 폴더에 "세금계산서_추출.xlsx"로 저장해줘.
```

> **실제 업무 팁**: 진짜 PDF 파일이 있으면 Claude에게 "이 PDF 읽어서 엑셀로 정리해줘"라고 시키면 `pdfplumber`를 설치해서 처리합니다.

---

## 데모 3: 자연어 데이터 분석 (7분)

> **학생이 느낄 것**: "SQL 몰라도 한국어로 물어보면 되네"

### Claude에게 이렇게 말하세요 (하나씩)

```
week05-hands-on/data/sales.db에서 매출이 가장 높은 법인 3개를 알려줘.
```

```
전월 대비 매출이 가장 많이 늘어난 법인과 월을 알려줘.
```

```
법인별 원화 환산 매출 합계를 보여줘. 최신 환율 기준으로.
```

```
6월 매출이 1월보다 줄어든 법인이 있어?
```

### 포인트
- 학생이 "나도 저렇게 물어보면 되는 거야?"라고 느끼게
- SQL이 화면에 나오지만 **학생이 SQL을 읽을 필요는 없다**는 걸 강조
- "여러분 엑셀에서 피벗 테이블 만들던 거, 한 줄이면 됩니다"

---

## 데모 4: 엑셀 서식 자동화 (5분)

> **학생이 느낄 것**: "보고서 포맷팅에 30분 걸리던 게 자동이네"

### Claude에게 이렇게 말하세요

```
week05-hands-on/data/sales.db 데이터로
"2026년 상반기 법인별 매출 보고서" 엑셀을 만들어줘.

요구사항:
- 시트1: 법인별 요약 (법인명, 통화, 외화합계, 원화환산) + 막대 차트
- 시트2: 월별 상세 피벗 (법인 × 월)
- 헤더는 짙은 남색 배경 + 흰 글씨
- 합계 행은 회색 배경 + 볼드
- 숫자는 천단위 콤마
- week07-hands-on/demos/output/ 폴더에 저장
```

---

## 데모 5: 데이터 정합성 체크 (5분)

> **학생이 느낄 것**: "감사 자료 대조를 눈이 아니라 코드가 하네"

### Claude에게 이렇게 말하세요

```
회계시스템과 ERP에서 뽑은 4월 매출 데이터를 비교하고 싶어.

먼저 샘플 데이터 2개를 만들어줘:
- 회계시스템_4월.xlsx: 법인 8개의 매출
- ERP_4월.xlsx: 같은 데이터인데 일부러 금액이 다른 게 2건, 누락된 게 1건

그다음 두 파일을 비교해서:
- 금액이 안 맞는 행
- 한쪽에만 있는 행
을 찾아서 week07-hands-on/demos/output/에 "정합성_체크결과.xlsx"로 저장. 불일치 행은 빨간 배경.
```

---

## 데모 6: 반복 보고서 템플릿 (5분)

> **학생이 느낄 것**: "매월 같은 양식에 숫자만 바꾸는 일이 끝났다"

### Claude에게 이렇게 말하세요

```
week05-hands-on/data/sales.db에서 2026년 4월, 5월, 6월
월별 매출 보고서를 각각 만들어줘.

각 보고서에 들어갈 내용:
- 타이틀: "월별 매출 보고서 — YYYY-MM"
- 작성일시 자동 표시
- 법인별: 당월 매출, 전월 매출, 증감, 증감률, 원화 환산
- 증감률은 퍼센트 서식 (증가 빨강, 감소 초록)
- 합계 행 (파란 배경)
- 하단에 작성/검토/승인 서명란
- 3개 파일을 한 번에 week07-hands-on/demos/output/ 폴더에 저장
```

## 강사 참고

### 시연이 잘 안 될 때
`demos/` 폴더에 정답 스크립트가 있습니다:
```bash
py demos/demo1_compare_excel.py
py demos/demo2_pdf_extract.py
# ... 등
```

### 시연 순서 추천
시간이 부족하면 **3 → 1 → 6** 순서를 추천합니다:
- 3 (자연어 분석): 가장 임팩트 큼, "SQL 몰라도 됨"
- 1 (엑셀 비교): 모든 재경팀이 하는 작업
- 6 (보고서 자동 생성): "매달 이거 하고 있었는데..."

### 학생에게 던질 질문
각 데모 후에:
> "이거 여러분 업무 중에서 비슷한 게 있나요?"

이 질문이 8주차 결과물로 연결됩니다.
